Спроектировать архитектуру системы SFMShop для высокой нагрузки.
Описать масштабирование, кэширование, балансировку нагрузки.

1) Горизонтальное масштабирование (несколько экземпляров приложения)
   Горизонтальное масштабирование реализуется путем подключения нескольких
   серверов и разворачиваением там экземпляров приложений таким образом
   таким образом запросы идут не на один сервер а распределяются по нескольким
   серверам, соответственно обрабатывается больше запросов
2) Кеширование (Redis) - Кешированияе - это временное хранение данных
   для быстрого доступ к ним. Redis - Это key-value база данных которая хранит данные
   непосредственно в оперативной памяти. Что бы каждый раз не запрашивать одни
   и те же данные мы помещаем данные в кеш на определенный промежуток времени.
   Что бы при следующем повторяющемся запросе взять данные из кеша.
3) Балансировка нагрузки (nginx) -
   это реализация механизма, который распределяет
   запросы между серверами. Типы балансировки:

- Round Robin - распределение запросов по очереди между серверами.
- Least Connection - запрос идет на тот сервер где меньше всего соединений
- IP Hash - запросы с конкретно IP адреса посылаются на один и тот же сервер

4) Масштабирование(Репликация) - это создание нескльких баз данных.
   Например запросы на получение идут на одну базу а на запись в другую
   Тем самым уменьшая нагрузку.

Метрики

1) Целевой RPS — это количество запросов, которое приложение
   должно обрабатывать в единицу времени. Чем больше значений
   RPS мы получаем, тем выше производительность нашего приложения.
2) Целевая latency - это время которе тратится на обработку одного запроса
3) Стратеги масштабирования - это механизмы которые позволяют адаптировать
   наше приложение под текущие условия.

- горизонтальное масштабирование - подключение нескольких серверов
- вертикальное масштабирование увеличение мощьности одного сервера
- Кеширование данных для быстрого доступа из кеша
- Создание реплик баз данных для разгрузки
- Балансировка - оптимазация входящих запросов от клиента

# Системный дизайн проекта SFMShop

### Компоненты:

1) Web-сервер: Обрабатывает входящие запросы и отвечает на них.
2) База данных: Хранит информацию о пользователях, продуктах и заказах.
3) API Gateway: Управляет входящими запросами между клиентом и различными микросервисами.

- Микросервисы:
    - user-service: Обрабатывает операции с пользователями (регистрация, аутентификация).
    - product-service: Обрабатывает операции с продуктами (добавление, удаление, поиск).
    - order-service: Обрабатывает операции с заказами (создание, обновление, отмена).
    - pyment-service: Обрабатывает операции с платежками(создание, возврат)

### Масштабирование:

- Вертикальное масштабирование: Увеличение мощностей серверов для обработки большего количества запросов.
    - Web-сервер: Увеличение RAM и CPU.
    - API Gateway: Использование кластеризации для распределения нагрузки.
- Горизонтальное масштабирование: Добавление доп. серверов для распределения нагрузки.
    - Web-серверы: Размещение на нескольких серверах и балансировка запросов с помощью API Gateway.
    - Микросервисы:  Использование микросервисной архитектуры для параллельного выполнения операций.

### Кэширование:

- Redis: Используется для кеширования часто запрашиваемых данных (например, информация о продуктах и пользователях).

## Метрики производительности

- CPU и память: Оптимизация использования ресурсов на серверах.
- Latency: Оценка времени для одного запрса.
- RPC - кол-во запросов за единуицу времени.
